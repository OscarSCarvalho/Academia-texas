# Guia de Contribuição

Obrigado por considerar contribuir para o SmartGym Admin! Este documento fornece diretrizes e instruções para contribuir com o projeto.

## 📋 Código de Conduta

Somos comprometidos em fornecer um ambiente acolhedor e inclusivo para todos.

## 🐛 Relatando Bugs

Antes de criar um relatório de bug, verifique a lista de issues, pois você pode encontrar uma já existente.

Ao criar um relatório de bug, inclua:
- Um título descritivo
- Descrição clara do comportamento esperado vs. atual
- Exemplos de código para demonstrar as etapas
- Ambiente: versão do Node.js, SO, navegador, etc.

## 💡 Sugerindo Melhorias

Sugestões de melhorias são bem-vindas! Ao criar uma sugestão de melhoria, inclua:
- Um título descritivo
- Descrição detalhada da melhoria sugerida
- Exemplos de como a melhoria funcionaria
- Possível implementação

## 🔧 Pull Requests

- Siga os padrões de código do projeto (TypeScript, ESLint)
- Inclua testes se aplicável
- Atualize a documentação conforme necessário
- Cada PR deve ter um escopo focado

### Processo
1. Fork o repositório
2. Crie uma branch para sua feature (`git checkout -b feature/AmazingFeature`)
3. Commit suas mudanças (`git commit -m 'Add some AmazingFeature'`)
4. Push para a branch (`git push origin feature/AmazingFeature`)
5. Abra um Pull Request

## 📝 Padrões de Código

- Use TypeScript com type safety completo
- Siga as regras do ESLint
- Use componentes do Shadcn UI quando possível
- Mantenha componentes pequenos e reutilizáveis

## 📚 Documentação

- Mantenha o README.md atualizado
- Adicione comentários para lógica complexa
- Documente componentes com props e exemplos

## ✨ Agradecimentos

Muito obrigado por suas contribuições! 🎉
