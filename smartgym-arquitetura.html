Visão Geral do Projeto
Objetivo, escopo e proposta de valor

O que é o SmartGym Control?

O SmartGym Control é uma plataforma SaaS (Software as a Service) completa para gestão de academias, desenvolvida para atender desde pequenas academias independentes até grandes redes e franquias com múltiplas unidades. A plataforma integra em um único ecossistema: controle de acesso por QR Code, gestão financeira com cobranças automáticas, aplicativo mobile para alunos, totem de autoatendimento, fichas de treino digitais, agendamento de aulas, dashboard gerencial com BI em tempo real e inteligência artificial para prevenção de churn (retenção de alunos).

Problema que Resolve
Academias utilizam múltiplos sistemas desconectados: um para financeiro, outro para treinos, outro para acesso — gerando retrabalho, inconsistência e perda de dados.

Uma única plataforma integrada que centraliza toda a operação da academia, desde o cadastro do aluno até a cobrança automática e controle de catraca.

Arquitetura do Sistema
Visão macro de todos os componentes e camadas

04
Banco de Dados & Estratégia de Dados
Multi-tenancy, cache, storage e mensageria
🐘
PostgreSQL 16 — Banco Principal
Hospedado no Supabase. Armazena todos os dados relacionais: tenants, alunos, planos, matrículas, pagamentos, acessos, treinos, agendamentos. Utiliza Row Level Security (RLS) para isolamento total entre tenants. Full-text search nativo com tsquery/tsvector.
⚡
Redis 7 — Cache & Tempo Real
Cache de sessões JWT, TTL de QR Codes (validade 24h por padrão), rate limiting por IP/tenant, Pub/Sub para WebSocket em tempo real (eventos de catraca, check-in ao vivo no painel), e como broker do BullMQ.
📦
S3 / Supabase Storage — Arquivos
Fotos de perfil dos alunos, documentos (RG, CPF, contrato), fichas de treino em PDF, laudos médicos, imagens de marketing. Acesso via URLs pré-assinadas com expiração, organizados por tenant_id/tipo/arquivo.
📬
BullMQ — Filas Assíncronas
Processamento assíncrono de: envio de e-mails em lote, cobranças automáticas mensais, push notifications agendadas, geração de relatórios pesados, inferência de IA (churn prediction), e webhooks de pagamento.

Estratégia Multi-Tenant — Row Level Security

-- Cada tabela possui tenant_id. A policy bloqueia acesso cross-tenant automaticamente. CREATE TABLE members ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), tenant_id UUID NOT NULL REFERENCES tenants(id), gym_unit_id UUID REFERENCES gym_units(id), name VARCHAR(200) NOT NULL, email VARCHAR(200) UNIQUE NOT NULL, created_at TIMESTAMPTZ DEFAULT now() ); -- Habilitar RLS na tabela ALTER TABLE members ENABLE ROW LEVEL SECURITY; -- Policy: cada sessão só enxerga dados do seu tenant CREATE POLICY tenant_isolation ON members USING (tenant_id = current_setting('app.tenant_id')::UUID); -- No backend (NestJS), antes de cada query: await prisma.$executeRaw`SET app.tenant_id = '${tenantId}'`;

Hierarquia de Dados — Multi-Tenant / Multi-Unidade

Tenant (Academia / Empresa) └── GymUnit (Unidade / Filial) ├── Member → Alunos matriculados na unidade ├── Instructor → Instrutores / Personal Trainers ├── Plan → Planos disponíveis (Mensal, Trimestral...) ├── Enrollment → Matrícula (Member + Plan + Datas) ├── Payment → Pagamentos e cobranças ├── AccessLog → Log de entradas/saídas ├── TrainingSheet → Fichas de treino └── Schedule → Aulas agendadas

06
Controle de Acesso — RBAC
Roles, permissões e hierarquia de usuários
O sistema utiliza Role-Based Access Control (RBAC) com 6 perfis distintos. Cada role define quais recursos e ações o usuário pode acessar, garantindo segurança e separação de responsabilidades.
👑 SuperAdmin
  • Acesso total à plataforma
  • Gerenciar todos os tenants
  • Configurações globais do SaaS
  • Monitoramento de infraestrutura
  • Suporte a clientes
🏢 GymOwner (Dono)
  • Todas as unidades do seu tenant
  • Configuração de planos e preços
  • Dashboard financeiro completo
  • Criar / gerenciar Managers
  • Relatórios comparativos
⚙️ Manager (Gerente)
  • Gestão da unidade vinculada
  • Gestão de alunos e matrículas
  • Relatórios da unidade
  • Criar instrutores e recepcionistas
  • Sem acesso financeiro global
🏋️ Instructor
  • Ver alunos sob sua supervisão
  • Criar e editar fichas de treino
  • Avaliações físicas
  • Agenda pessoal
  • Sem acesso financeiro
🖥️ Receptionist
  • Check-in manual de alunos
  • Cadastro de novos alunos
  • Renovação de planos
  • Ver agenda do dia
  • Sem relatórios financeiros
🎓 Student (Aluno)
  • App mobile apenas
  • Ver própria ficha de treino
  • QR Code de acesso
  • Histórico de pagamentos
  • Agendar aulas em grupo

09
Segurança & Conformidade
LGPD, OWASP, criptografia e boas práticas
🔒
Autenticação & Autorização
JWT com expiração curta (15min) + Refresh Token rotativo (7 dias).
2FA via TOTP para accounts sensíveis (GymOwner, Manager).
Bloqueio após 5 tentativas falhas (brute-force protection).
OAuth2 com PKCE para app mobile.
RBAC granular por recurso e ação.
🛡️
OWASP Top 10
Prevenção de SQL Injection via ORM (Prisma — queries parametrizadas).
XSS prevention via sanitização e Content-Security-Policy.
CSRF tokens em formulários sensíveis.
Rate limiting por IP e por tenant no API Gateway.
Helmet.js (NestJS) para headers de segurança HTTP.
🔐
Criptografia & Dados
Senhas hasheadas com bcrypt (salt rounds: 12).
Dados sensíveis (CPF, dados bancários) criptografados em repouso (AES-256).
HTTPS/TLS 1.3 em todas as comunicações.
Arquivos privados acessíveis apenas via URL pré-assinada (expiração curta).
⚖️
LGPD — Lei Geral de Proteção de Dados
Consentimento explícito no cadastro do aluno.
Direito de exclusão: endpoint para "esquecimento" de dados pessoais.
Exportação dos dados do aluno em formato JSON/PDF.
Audit log: toda alteração de dados pessoais é registrada com quem fez e quando.
DPO (Data Protection Officer) configurável por tenant.
